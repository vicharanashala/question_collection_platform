## RazorpayX Payouts - Key Learnings

### Mock UPI IDs for Testing
Any structurally valid UPI ID format works for test payouts:
- `test@upi`, `success@upi`, `randomfarmer@ybl`, etc.
- RazorpayX accepts any `something@something` format — it doesn't validate the actual UPI handle.

### Payment Gateway vs. RazorpayX Payouts
- **Collecting money (Payment Gateway):** `success@razorpay` / `failure@razorpay` auto-pass/fail in UI
- **Sending payouts (RazorpayX):** Any mock UPI ID accepted; actual bank transfer simulated via RazorpayX Test Dashboard

### Testing Payout Flow Locally
1. `POST /v1/payouts` with customer identifier (e.g. `2323230071156695`)
2. Status instantly moves to `processing` in DB + backend console
3. Open **RazorpayX Test Dashboard → Payouts** → click transaction → **"Move Payout to Next State"** → choose `processed` (success) or `reversed` (failure)
4. This fires the local webhook script to process the state change
